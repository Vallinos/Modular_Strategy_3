Accessibility Web Design

For this project I am attempting to make a web application page that is more accessible to people of varied needs. As a result, this page would be used to showcase a number of methods that would allow for more accessibility. This page will focus on HTML, Python, Flask, and CSS.

The following is what I researched to make for a more accessible web page:

Visual Impairment

For example, I put a slider that would help users with potential visual impairments by providing options for text size.
(WCAG 2: 1.4.4)
This gives users the ability to choose between a few font sizes, ranging from 20px to larger sizes for users to more easily read the text.

The colors should also remain consistant throughout the web page so as to not disorientate users while they use the application, text will remain one certain color, links will be a certain color, and the headers should remain consitant. For this web application the text will be blue, the links will be red, and the headers will be green. Why would I choose these strange colors you might ask. It is to move on to the next point.
(WCAG 2: 1.4.1)

There are a number of people in the world who have some sort of color vision deficiency. About 8% of men and .4-.5% of women report some deficiency in green/red vision. There are also individuals in rare cases, about .05% of people, who have deficiency in seeing blue. Additionally, there are common groupings of colorblindness that are commonly seen, those being protanopia, deuteranopia, and tritanopia.


(colorlitelens)
(davidmathlogic)

With this being the case, I have chosen to include additional buttons to switch to a color vision contrast to specifically contrast for protanopia colorblindness that would convert all text, headers, and links to a contasted colorscheme. Text would be a different shade of blue, header 2 would be orange to contrast the default green while header 1 would be a shade of violet, red will also be another shade of red. The background changes from tan to a more pinkish tan as well. These would create better contrast for protanopia. The option to switch would be at the top.

Accessible Framework



I have made sure that each link is clearly visible by giving them a distinct color making it easier to spot for the user.
In addition to buttons being clearly labeled at the top of the web application, there should be buttons placed at the bottom of the application that are clearly visible to the user. An important option to be provided for the user is to be able to ascend back to the top of the web application. Pressing the button labeled "Back to Top" would take the user back to the top to be used at the user's discretion. There is also a scrolling back to top bottom that moves along with the user as they scroll down the screen to go back to the top at any point if necessary.
(WCAG 2: 2.4)
The other buttons, to simplify the web application's navigation to the user would take them to the other two implementations as well as the landing page.
